Expand description

As a simulation runs, different pieces emit Events. The Analytics object listens to these, organizing and storing some information from them. The UI queries Analytics to draw time-series and display statistics.

For all maps whose weekday scenario fully runs, the gameโs release includes some โprebaked results.โ These are just serialized Analytics after running the simulation on a map without any edits for the full day. This is the basis of A/B testing โ the player can edit the map, start running the simulation, and compare the live Analytics to the prebaked baseline Analytics.
